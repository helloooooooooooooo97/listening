# v0.2.6 Product Plan — 今日听词 + 单词板块重构

> **核心理念变化：** 所有单词只来源于你听过的音频，不依赖外部词库。
> 今天的单词 = 你今天听音频时遇到的所有单词，自动收集，独立复习。

## 二、核心概念：听音得词

### 用户场景

1. **场景 A：今日听过即所得**
   - 今天听了 3 段雅思听力
   - 打开"今日单词"→ 自动列出这 3 段音频涉及的所有单词（去重）
   - 每个词标记：来自哪个音频、在音频中出现的时间点
   - 看到 » 今天累计听过 42 个不同单词

2. **场景 B：听完立即复习**
   - 听完一段音频，底部自动显示"本课单词"
   - 点击"复习本课单词" → 进入单词复习卡片
   - 逐词回顾：显示词 → 回想 → 显示中文释义 → 自评

3. **场景 C：逐个点击听原声**
   - 单词列表里点击任意单词 → 右侧详情面板
   - 面板列出该词在每节课的每个出现时间点
   - 点击时间戳 → 跳转到音频精确位置播放

4. **场景 D：今日单词集中复习**
   - 在"今日单词"专区点击"开始今日复习"
   - 系统按频率排序，逐个展示今日遇到的单词
   - 复习完标记"已掌握"或"需复习"

## 三、数据库 — 听词记录表

### 3.1 新增表

```sql
-- 实际听过的单词（前端按播放区间筛选后上报）
CREATE TABLE IF NOT EXISTS listened_words (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    word TEXT NOT NULL,
    audio_id TEXT NOT NULL,
    audio_title TEXT NOT NULL,
    listened_date TEXT NOT NULL,
    created_at TEXT DEFAULT (datetime('now'))
);
CREATE INDEX IF NOT EXISTS idx_lw_date ON listened_words(listened_date);
CREATE INDEX IF NOT EXISTS idx_lw_word ON listened_words(word);
CREATE INDEX IF NOT EXISTS idx_lw_audio ON listened_words(audio_id);
CREATE UNIQUE INDEX IF NOT EXISTS idx_lw_unique ON listened_words(word, audio_id, listened_date);
```

### 3.2 数据产生流程

```
1. 用户播放音频 → trackPlay() 记录起始时间
2. 暂停/停止 → flushTrack() 算出 elapsed 秒数
3. 前端从当前 lesson.words 筛选 word.start <= elapsed 的单词
4. 去重后 POST /api/progress/listened-words 发给后端
5. 后端存入 listened_words 表（INSERT OR IGNORE）
```

### 3.3 已有表关系

| 表 | 作用 | 来源 |
|----|------|------|
| `word_occurrences` | 单词在哪些音频的哪些时间点出现 | 课程 JSON 解析 |
| `play_history` | 用户何时播放过哪些音频 | 播放行为 |
| `listened_words` | **实际听到的单词（按时间区间筛选）** | 前端上报 |
| `word_progress` | 单词掌握状态 | 用户复习行为 |

## 四、"今日单词" — 核心功能 P0 ⭐

### 4.1 今日单词列表

**查询逻辑：**
```sql
-- 获取今日听过的所有单词（去重，按频率排序）
SELECT dlw.word, COUNT(DISTINCT dlw.audio_id) AS audio_count,
       wp.known, wp.last_score, wp.reviewed_at
FROM daily_listened_words dlw
LEFT JOIN word_progress wp ON wp.word = dlw.word
WHERE dlw.listened_date = date('now')
GROUP BY dlw.word
ORDER BY audio_count DESC, dlw.word ASC
```

- 显示今天听过的所有单词
- 每个词显示：单词、来自几段音频、掌握状态
- 按频率排序（跨多个音频的词排前面）

### 4.2 今日单词语音关联

- 点击单词 → 右边显示该词在**今天听的音频中**的所有出现时间点
- 点击时间戳 → 跳转到对应音频播放
- 整合 `word_occurrences` + `daily_listened_words` 双表 JOIN

### 4.3 今日单词复习

- 进入"今日单词"专区 → 点击"开始复习"
- 复习流程（flashcard 风格）：
  1. 显示单词 + 音频标题来源
  2. 用户在心里回想
  3. 点击显示答案（中文释义 + 原句片段）
  4. 三级自评：忘记(0分) / 模糊(50分) / 想起(100分)
  5. 记录到 `word_progress` 表
  6. 自动切下一个词
- 复习结束后显示**今日复习总结**：
  - 今日共遇到 N 个单词
  - 已复习 M 个
  - 掌握率 X%
  - 继续鼓励："还需复习 K 个未掌握单词"

## 五、WordsView 重构 — 排版与逻辑优化 P0 ⭐

### 5.1 布局重构

**现状问题：** 单词以网格排列，信息密度低，无分类层级。
**改为：**

```
┌──────────────────────────────────────────┐
│  单词                              [搜索]  │
│                                          │
│  ┌──────────────┬──────────────────────┐ │
│  │ 📅 今日单词    │ 📚 所有单词           │ │
│  │ 今天听到 42 个  │ 历史累计 1024 个      │ │
│  │ 已复习 12 个    │ 已掌握 520 个        │ │
│  │ [开始今日复习]  │                      │ │
│  ├──────────────┼──────────────────────┤ │
│  │ 📖 按音频来源   │                      │ │
│  │ IELTS S1 … 15词 │                      │ │
│  │ Aesop Fables   │                      │ │
│  │   The Fox … 8词 │                      │ │
│  └──────────────┴──────────────────────┘ │
└──────────────────────────────────────────┘
```

### 5.2 分类筛选栏（顶部 tabs）

```
[📅 今日单词] [📚 全部单词] [📖 按音频] [🔄 待复习] [✅ 已掌握]
```

- **今日单词** — 默认选中（如果有今日听词记录）
- **全部单词** — 所有历史出现单词
- **按音频** — 按音频分组展示（类似文件夹结构）
- **待复习** — `word_progress` 中 last_score < 60 或未标记已知的词
- **已掌握** — 标记为已知的词

### 5.3 单词列表项优化

```
┌──────────────────────────────────────────┐
│  abandon       🎧 3段音频  [✓已掌握]  ♡  │
│  ability       🎧 1段音频  [🆕未复习]      │
│  absolutely    🎧 2段音频  [🔄待复习]  ♡  │
│  ...                                      │
└──────────────────────────────────────────┘
```

- 单词左对齐，更清爽
- 左侧显示频率条/小徽章（跨几个音频）
- 右侧显示掌握状态标识 + 收藏按钮
- hover 显示"播放"按钮
- 已掌握的词灰显处理

### 5.4 无限滚动优化

- 保留现有的 callback ref 无限滚动
- 增加"回到顶部"浮动按钮
- 增加加载中的骨架屏效果（取代简单旋转动画）

### 5.5 筛选面板重做

- 从下拉菜单改为抽屉式侧边栏或底部 sheet
- 筛选维度：
  - **时间：** 今天 / 本周 / 本月 / 全部
  - **来源：** 按音频筛选
  - **状态：** 全部 / 未复习 / 已掌握 / 待复习
- 每个筛选项显示对应词数
