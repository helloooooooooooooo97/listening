# v0.1.4 产品计划 — 听写模式嵌入播放详情页

**日期**: 2026-06-07
**主题**: 听写不再独占全屏，嵌入播放详情页左侧，右侧侧边栏保持可见。

---

## 一、现状问题

```
当前：听写激活 → ContentPanel 直接返回 <DictationView />
      ↓
     整个页面被替换，看不到 transcript、侧边栏、标题栏
```

## 二、目标布局

```
┌──────────────────────────────────────────┐
│  ← 标题          📝(高亮)  ➕  🔄       │  Header 不变
├──────────────────────────────────────────┤
│  ┌──────────────────┬──────────────────┐ │
│  │ 听写 · 句子 3/12 │ 片段 听写 收藏   │ │  ← 侧边栏保持
│  │ 均分 85%         │                  │ │
│  │                  │  ┌──────────────┐ │ │
│  │ [▶] 输入你听到的 │  │ 句子1  90%   │ │ │  ← 可切换到听写tab
│  │ [____________]   │  │ 句子2  75% ✗ │ │ │    查看历史/复练
│  │ [提交] [跳过]    │  │ 句子3  ...    │ │ │
│  │                  │  └──────────────┘ │ │
│  │ ← 句子选择器 →   │                  │ │
│  └──────────────────┴──────────────────┘ │
└──────────────────────────────────────────┘
```

## 三、改动点

### 3.1 ContentPanel — 移除全局拦截
- [ ] 删除 `if (dictationActive) return <DictationView />`
- [ ] `DictationView` 改为普通组件，由 PlayerBar 按需渲染

### 3.2 PlayerBar 展开视图 — 集成听写
- [ ] 展开视图中判断 `dictationActive`
- [ ] dictationActive = true 时，左侧渲染听写 UI（替代 TranscriptView）
- [ ] 右侧 PlaybackDetailTabs 保持可见
- [ ] 标题栏 📝 按钮高亮表示听写中，点击退出

### 3.3 听写 UI 适配 — 拆分为嵌入式组件
- [ ] 从 DictationView 抽取核心逻辑为 `EmbeddedDictation` 组件
- [ ] 接收 `lesson`（当前音频）作为 prop，不再从 audioStore 推断
- [ ] 句子选择器水平排列在 typing/feedback 区域上方
- [ ] 保持现有交互：自动播放句子、提交对比、分数反馈

### 3.4 听写 ↔ 侧边栏联动
- [ ] 侧边栏听写 tab 实时显示当前进度
- [ ] 点击侧边栏听写记录 → 跳转到对应句子并进入听写模式
- [ ] 完成全部句子后在原位置显示 CompletionScreen
