# v0.1.0 产品计划 — 交互细化 & 体验流畅性

**日期**: 2026-06-07
**主题**：零新功能，打磨每一个交互细节，让已有功能用起来「舒服」。

---

## 一、设计目标

> 不是「能不能用」，而是「好不好用」。所有改动必须是用户可感知的体验提升。

| 维度 | 当前问题 | 目标 |
|---|---|---|
| **即时反馈** | 点击/操作后缺乏即时响应，用户不确定操作是否生效 | 每次操作 ≤100ms 内有视觉反馈 |
| **动效流畅** | 页面切换生硬、列表删增突兀、高亮跳跃 | 所有状态变化有过渡，60fps |
| **操作精度** | 波形点击偏移、拖拽选区不精准、进度条难微调 | 关键操作有吸附/放大辅助 |
| **连续性** | 句子间切换有停顿感、循环边界生硬、模式切换中断 | 播放体验无断裂感 |
| **触摸体验** | 移动端交互仍是桌面思维，滑动/长按未利用 | 移动端手势原生感 |

---

## 二、详细任务

### 🔴 P0 — 播放体验流畅性

> 播放是核心中的核心，任何卡顿、延迟、断裂都会被放大。

#### 2.1 播放/暂停响应优化

- [ ] **按下即反馈**：Play/Pause 按钮改为 `onPointerDown` 触发状态切换（而非 `onClick` 等抬起），消除 ~100ms 延迟感。按钮按下瞬间图标切换 + 缩放回弹。
- [ ] **播放启动延迟**：当前 `playLesson()` 内有 `waitForReady` + `saved position` seek 逻辑，音频未就绪时用户点播放无反应。改为：点击瞬间按钮进入「准备中」动画（脉冲），就绪后自动播放。
- [ ] **Seek 拖动体验**：PlayerBar 进度条改为可拖动滑块（当前仅波形支持点击 seek）。拖动时跟随显示时间预览浮泡，松手瞬间 `currentTime` 跳转，音频无缝衔接。

#### 2.2 逐词高亮跟随

- [ ] **高亮过渡动画**：当前词级高亮是瞬间切换（`activeWordId` 突变 → className 突变）。改为 CSS `transition` 做 80ms 的淡入淡出，让高亮「滑」过去而非「跳」过去。
- [ ] **自动滚动平滑度**：当前 `activeLineRef.scrollIntoView({behavior:'smooth', block:'center'})` 在快速播放时滚动过于频繁。改为：仅在当前活跃行超出可视区域 2 行以上时才触发滚动；滚动动画用自定义 `scrollTo` 替代原生 smooth（可控速）。
- [ ] **句子切换呼吸感**：逐句循环模式下，当前句子播完 → 跳回句首时加 150ms 微停顿 + 视觉闪烁提示（句子行短暂高亮），让用户感知循环边界。

#### 2.3 变速播放平滑

- [ ] **变速切换无爆音**：当前 `setRate` 直接改 `playbackRate`，部分浏览器在切换瞬间有 audible pop。改为：切换前瞬间静音(volume→0)→ 改 rate → 20ms 后恢复音量，做微型 crossfade。
- [ ] **变速指示动画**：变速按钮点击后，速度数字做一次弹性缩放（scale 1→1.3→1），让用户感知速度已切换。

#### 2.4 波形交互细化

- [ ] **Hover 时间预览**：鼠标悬停波形时，跟随光标显示当前指向的时间戳浮泡（`1:23.4`）。
- [ ] **点击精度优化**：在波形上 mousedown → mousemove（< 5px 偏移）→ mouseup 才算 seek，避免误触。同时点击位置做微弱涟漪动画。
- [ ] **片段选区可视化**：在波形上拖拽可直接创建片段选区（当前仅在 TranscriptView 支持）。选中区域高亮拖动边缘可微调起止点。

### 🟡 P1 — 听写模式交互打磨

> 听写是最重交互的模式，当前流程是「能用」但远不到「好用」。

#### 3.1 输入体验

- [ ] **输入框自动聚焦 + 光标**：进入新句子时输入框自动 `focus()` 已做。补充：聚焦时输入框边框从灰色 → accent 色做 300ms 过渡，光标闪烁醒目。
- [ ] **播放-输入联动**：当前需要手动点播放按钮。改为：进入新句子自动播放一遍（已有），但增加「播完自动聚焦输入框 + 屏幕键盘弹出（移动端）」。
- [ ] **重听按钮计数**：「重听」按钮旁显示当前句子已播放次数（如 `🔊×2`），帮助用户管理听力策略。
- [ ] **快捷操作**：
  - `Enter` 提交（已有）
  - `Ctrl+Enter` 跳过
  - `Ctrl+←/→` 在已输入的单词间跳转（词级光标移动）
  - 听写中按 `Space` 重播当前句（不提交）

#### 3.2 反馈阶段动效

- [ ] **词级评分 staggered 入场**：提交后，每个词的判定结果（正确/错误/遗漏/多余）依次弹出，间隔 30ms（当前已有 `animationDelay`，但仅为 `scale-in`）— 增加色彩从灰→目标色的过渡。
- [ ] **分数揭晓动效**：分数数字从 0 滚动到实际分数（count-up animation），而非直接显示静态数字。
- [ ] **错误词高亮回看**：Feedback 阶段，TranscriptView 中对应的句子自动滚动到视野内，错误词做红色波浪下划线闪烁两次，随后保持静态下划线。用户点击错误词可跳转播放该词位置。
- [ ] **正确率变化反馈**：如果本次得分比上次高 → 分数显示绿色 + 一个微小的 👍 弹跳；比上次低 → 黄色，鼓励继续。

#### 3.3 句子导航

- [ ] **句子选择器增强**：当前 `SentenceSelector` 显示句子编号圆点。改为：每个圆点内含微型进度环（该句听写过几次/均分颜色），一眼可见哪些句子弱。
- [ ] **完成页面对比动画**：完成屏幕的每句分数标签按顺序弹出，分数从低到高排列，最低分句子高亮提示「建议重练」。

### 🟡 P1 — 页面过渡与导航

> 当前所有 View 切换都是瞬间替换，毫无过渡，体验生硬。

#### 4.1 View 切换动画

- [ ] **淡入淡出切换**：`ContentPanel` 内不同 View 切换时，旧 View 做 150ms fade-out，新 View 做 200ms fade-in + slide-up(8px)，交叉过渡。
- [ ] **侧边栏选中态过渡**：当前选中项背景色瞬间切换。改为：选中指示器（背景色块）做 position 动画，从一个导航项滑到另一个，类似 iOS Settings 的选中效果。

#### 4.2 移动端 Tab Bar

- [ ] **Tab 切换图标动画**：选中 Tab 的图标做弹性缩放 + 色彩过渡（当前仅静态高亮）。
- [ ] **底部安全区适配**：PlayerBar 和 TabBar 的 `safe-area-inset-bottom` 已处理，补充：横屏时自动隐藏 TabBar（横屏空间宝贵）。

#### 4.3 阅读视图（Expanded PlayerBar）

- [ ] **展开/收起手势**：当前点击标题区域展开全文阅读视图。改为：PlayerBar 区域上滑手势展开（类似 Apple Music 的 Now Playing），下滑收起。动画用弹簧曲线。
- [ ] **展开过渡动画**：当前 `transition-all duration-300` 仅对 PlayerBar 自身有效。Transcript 内容区域在展开时做 200ms 延迟淡入，收起时瞬间隐藏，避免内容跳变。

### 🟢 P2 — 列表 & 收藏交互

#### 5.1 列表操作反馈

- [ ] **删除动画**：收藏列表/片段列表中删除一项时，该项做高度收缩 + 淡出（200ms），后续项平滑上移填补空位。
- [ ] **收藏切换动效**：点击 ♥ 收藏时，图标做弹性缩放 + 短暂的颜色脉冲（从 accent 色扩散一圈涟漪），类似 Twitter 的 like 动画。
- [ ] **加入队列反馈**：点击「添加到队列」后，按钮短暂变为 ✓ 确认态（1s）+ Toast，当前仅 Toast。
- [ ] **列表排序拖拽**：PlaylistView 中的队列项支持拖拽排序（当前仅 `reorder` 方法但无可交互 UI）。

#### 5.2 空状态与加载

- [ ] **首次加载过渡**：App 启动时 `lessons` 为空数组，首页直接显示 0 个音频。改为：音频 Grid 区域显示 8 个卡片骨架屏，数据到达后骨架屏依次消散（staggered reveal）。
- [ ] **空列表引导**：当某列表为空（如暂无片段、暂无收藏），显示简洁插画 + 一句引导文案（如「选中文字拖拽即可创建片段」），而非空白页面。

---

### 🟢 P2 — 移动端触摸体验

#### 6.1 手势支持

- [ ] **列表项滑动操作**：收藏列表/片段列表支持左滑露出删除按钮（iOS 风格），而非只能进详情页点删除。
- [ ] **波形双指缩放**：在波形区域双指捏合可缩放时间精度（横轴拉伸），方便在大音频中精确定位。
- [ ] **长按单词**：Transcript 中对单词长按（> 300ms）弹出上下文菜单：播放此词 / 查词 / 标记已知 / 收藏。

#### 6.2 触觉反馈

- [ ] **Haptic 反馈**（iOS/Android 支持时）：播放/暂停切换、收藏切换、听写提交、片段保存等关键操作触发轻微震动。
- [ ] **边界触觉**：循环播放跳回句首、Seek 到 0:00 或结尾时轻震提示。

## 四、验收 Checklist

### 播放体验
- [ ] 点播放按钮 → 按下瞬间有视觉反馈 → 100ms 内开始出声（或显示加载动画）
- [ ] 拖动进度条 → 时间浮泡实时跟随 → 松手立即跳转无沉默
- [ ] 逐词高亮平滑跟随，无视觉跳跃
- [ ] 变速切换无爆音，有速度切换动画
- [ ] 波形悬停显示时间，点击精准无偏移
- [ ] 循环模式边界有感提示

### 听写体验
- [ ] 进入听写 → 自动播放 → 播完自动聚焦输入框
- [ ] 提交后分数滚动揭晓，词级结果 staggered 出现
- [ ] 错误词在 Transcript 中高亮闪烁
- [ ] 句子选择器可见每句掌握程度
- [ ] 完成页有弱句提示

### 导航体验
- [ ] View 切换有 200ms 淡入效果
- [ ] 侧边栏选中态平滑滑动
- [ ] 移动端 Tab 切换有弹性动画
- [ ] 阅读视图展开/收起有弹簧手势感

### 列表体验
- [ ] 删除有收缩动画
- [ ] 收藏 ♥ 有弹性缩放 + 涟漪
- [ ] 加载中有骨架屏 staggered 消散
- [ ] 空列表有引导文案

### 移动端
- [ ] 列表项支持左滑操作
- [ ] 关键操作有触觉反馈（支持的设备）
- [ ] 长按单词有上下文菜单
- [ ] 波形支持双指缩放
