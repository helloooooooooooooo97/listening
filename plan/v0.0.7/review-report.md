## 🔍 关键设计决策

### 1. CSS 变量 vs Tailwind 主题
选择 CSS 自定义属性而非 Tailwind `dark:` 变体。原因：
- 现有代码大量使用行内样式和任意值（`bg-[#0a0a0b]`），CSS 变量可直接替换
- 运行时切换无需 Webpack/Vite 重新编译
- 与 Canvas 2D 上下文兼容（通过 `getComputedStyle` 解析）

### 2. 多维度交互标记
每个单词使用不同视觉通道展示交互痕迹，而非优先级折叠：
- 背景色 → 片段
- 文字色 → 已掌握
- 装饰线 → 听写错误
- 行内图标 → 收藏
- 尾部徽章 → 句子分数

### 3. 全屏覆盖层架构
覆盖层 `z-30` + 底部播放栏 `z-40`，覆盖层底部留 72px padding
- 覆盖层全屏（侧边栏被遮住）
- 底部播放栏保持可见并直接复用原组件
- 展开时底部栏 left-0 全宽，收起时 left-56 恢复
