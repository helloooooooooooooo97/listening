# 歌词阅读界面交互设计计划 V2

> 核心原则：**每种痕迹使用独立的视觉维度，互不冲突、无需优先级取舍。**

---

## 1. 视觉维度分配

每个单词本身就是一个「画布」，不同痕迹使用不同的视觉通道：

```
     ┌──────────────────────────────┐
     │        视觉维度矩阵            │
     ├────────────┬─────────────────┤
     │ 背景色      │ 片段深度 (黄)    │
     │ 文字色      │ 已掌握 (绿色)    │
     │ 下划线      │ 听写错误 (红波)  │
     │ 左侧图标    │ 收藏 (♥)        │
     │ 尾部标记    │ 句子听写分数     │
     └────────────┴─────────────────┘
```

### 互不冲突的原因

| 维度 | 影响范围 | 与谁重叠 |
|---|---|---|
| 背景色 | 整个词背景 | 独占背景通道 |
| 文字色 | 字的颜色 | 独占文字颜色通道 |
| 下划线 | 词下方 | 独占装饰通道 |
| 角标 | 右上角 6x6px | 迷你空间，一个符号足够 |
| 尾部标记 | 句子末尾 | 句子级，不影响词 |

## 2. 单词级痕迹（可同时存在）

```
                      ♥        ← 收藏（右上角标）
                      ↓
            ┌──────────────┐
 已知 →     │   grapes      │  ← 绿色文字 tint
            │   ──────      │
            └──────────────┘
                 ↓
         红色波浪下划线     ← 听写错误（装饰线，不占空间）

片段 → 黄色背景（透明度累积） ← 背景色通道
```

### 2.1 「已掌握」— 文字色通道
```
实现：text-emerald-300/70（绿色半透明）
效果：单词略微发绿，不刺眼
不冲突原因：仅改变文字颜色，不影响其他维度
```

### 2.2 「已收藏」— 词前左侧图标
```
实现：HiHeart 组件 size={8}，单词左侧
位置：单词 inline 前面，加 margin-right: 1px
不冲突原因：占据单词左侧间隙，不影响词本身的背景/文字色/下划线
```

### 2.3 「在片段中」— 背景色通道
```
实现：rgba(250,204,21, α) 黄底
累积：1次=0.06 / 2次=0.12 / 3次+=0.2
不冲突原因：背景色独立通道，与文字色/下划线/角标无关
```

### 2.4 「听写错误」— 下划线装饰通道
```
实现：border-bottom: 2px wavy rgba(239,68,68,0.5)
效果：红色波浪线，类似拼写检查
不冲突原因：文字下方装饰，不影响背景/文字色/角标
```

## 3. 句子级痕迹

### 3.1 听写分数 — 句子尾部徽章
```
 1  0:05  The fox was very hungry...     [78%]
                                     ↑ 灰色徽章，hover显示详情
```

### 3.2 片段数量 — 句子头部小标记
```
● 1  0:05  The fox was very hungry...
↑ 绿点 = 该句有片段
```

## 4. 叠加效果示例

```
   ♥grapes           ♥looked            very
   ──────              ──────
   ↑ 红波              ↑ 红波
   黄底(1次)           黄底(2次)
   绿色文字            绿色文字
```

| 词 | 表示含义 |
|---|---|
| `♥grapes` 绿字 + ♥图标 + 黄底 + 红波 | 已掌握·已收藏·在片段中·听写错 |
| `♥looked` 绿字 + ♥图标 + 更深黄底 + 红波 | 同上，但在2个片段中 |
| `very` 默认白字 + 红波 | 仅听写错误 |
| `the` 默认白字 | 无任何交互记录 |

所有标记共存，无需优先级取舍。

## 5. CSS 实现方案

```css
/* 已掌握 — 绿色文字 */
.word-known {
  color: rgba(16, 185, 129, 0.75);
}

/* 片段中 — 黄色背景累积 */
.word-clipped-1 { background: rgba(250, 204, 21, 0.06); }
.word-clipped-2 { background: rgba(250, 204, 21, 0.12); }
.word-clipped-3 { background: rgba(250, 204, 21, 0.2);  }

/* 听写错误 — 红色波浪下划线 */
.word-dictation-error {
  border-bottom: 2px wavy rgba(239, 68, 68, 0.5);
}

/* 收藏 — 词前左侧图标 */
.word-favorited::before {
  content: '♥';
  display: inline;
  font-size: 7px;
  color: rgba(250, 45, 72, 0.7);
  margin-right: 1px;
}
```

## 8. 关键设计决策

> **不设优先级、不互斥、不折叠。**
> 每个维度（背景/文字色/下划线/角标）各自独立，同时存在时视觉清晰可辨。
> 比「优先级折叠」方案更简单、更直观、实现成本更低。
