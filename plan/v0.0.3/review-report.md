## 二、功能评审

### ✅ 做得好的

1. **Apple Music 布局** — 左侧窄导航（首页/课程/片段/单词/最近播放），右侧大内容区，底部常驻播放栏，结构清晰
2. **全局音频状态** — Zustand `audioStore` 单例 Audio 元素，所有组件共享播放状态，点击即播
3. **底部播放栏常驻** — 无内容时显示"未在播放"，控件半透明禁用，始终可见
4. **全屏歌词覆盖** — 点击底部栏封面展开全屏歌词（TranscriptView），带词级高亮和时间同步
5. **首页仪表盘** — 全局搜索 + 5 项统计卡片（课程/片段/句子/单词/总时长）
6. **片段收藏** — 拖拽选词保存 + 全局 Zustand Store 同步，修复了 v0.0.2 的多组件状态隔离 Bug
7. **单词面板** — 去重统计 + 按频率排序标签云 + 点击查看详情（次数、所在课程）
8. **循环模式** — 全部/单句/片段三种模式，底部栏图标切换 + tooltip
9. **键盘快捷键** — Space/←→/↑↓/R/1-5/S 全局快捷键，输入框聚焦时自动禁用
10. **音频预加载** — App 启动时后台预加载全部课程音频，降低首次播放延迟

### ❌ 缺失的核心功能

| 功能 | 优先级 | 影响 |
|---|---|---|
| 听写模式 | 🔴 高 | 产品计划阶段7，0进度 |
| 跟读模式 | 🟡 中 | 产品计划阶段8，0进度 |
| 学习进度追踪 | 🟡 中 | 完成课程数/听力时长/准确率无持久化统计 |
| 课程封面自定义 | 🟢 低 | 当前使用固定渐变，不支持自定义图片 |

### 🐛 已知问题

1. **课程卡片点击即播放** — 交互符合预期，但没有"仅查看"的入口
2. **歌词覆盖层缺少关闭动画** — 全屏歌词展开/收起无过渡动画
3. **底部栏按钮紧凑** — 小屏幕下按钮间距过密
4. **单词面板首次加载慢** — 需要 fetch 全部课程 JSON 后才能显示
5. **无课程导入 UI** — 添加新课仍需手动运行 WhisperX 脚本 + 复制文件
6. **无错误重试** — API 请求失败静默处理，用户无感知

## 四、架构评审

### ✅ 优点

1. **Zustand Store 分层** — `audioStore`（播放）、`clipsStore`（收藏）、`settingsStore`（记忆）、`toastStore`（通知），职责分离
2. **ContentPanel 统一渲染** — 通过 `activeSection` 切换视图，避免路由复杂度
3. **预加载池设计** — `preloadLessonAudio()` 利用浏览器缓存
4. **后端 stats 端点** — `GET /api/lessons/stats` 返回去重统计，避免前端全量加载
5. **组件精简** — Sidebar(52px) + ContentPanel(flex-1) + PlayerBar(fixed) 三层结构

### ⚠️ 风险

1. **单词面板全量加载** — 首次打开单词页 fetch 全部课程 JSON，课程增多后性能下降
2. **ContentPanel 过大** — 单一组件包含首页/课程/片段/单词/最近 5 个视图，超过 500 行
3. **无后端缓存** — stats 端点每次重新遍历全部课程文件
4. **无错误边界** — API 请求失败静默处理

## 五、v0.0.4 建议

1. **听写模式 MVP** — 隐藏文本，逐句播放，输入框对比原文
2. **ContentPanel 拆分** — 首页/课程/片段/单词/最近各拆为独立组件
3. **歌词覆盖层动画** — 展开/收起过渡动画

### 🟡 P1 — 应该做

4. **课程导入 UI** — 上传音频 → 自动运行 WhisperX → 生成 JSON

8. **移动端响应式** — 屏幕 < 768px 时侧边栏折叠为底部 Tab

### 🟢 P2 — 可以做

12. **i18n 国际化** — 英文界面选项
