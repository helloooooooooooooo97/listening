# v0.1.1 产品计划 — 播放详情页交互打磨

**日期**: 2026-06-07
**主题**: 继续打磨播放详情页，聚焦 transcript ↔ 侧边栏的联动交互。

---

## 一、当前状态

播放详情页已有左右分栏布局：
- 左侧：逐句文本（TranscriptView）
- 右侧：tabs（片段 / 听写 / 收藏）

**存在的问题**：
- 侧边栏和文本各自独立，缺乏联动
- 点击片段后只是播放，transcript 不滚动到对应位置
- 听写记录点击不跳转句子
- 没有「当前正在播放哪句/哪个片段」的视觉反馈
- 移动端侧边栏未适配

---

## 二、详细任务

### 🔴 P0 — 文本 ↔ 侧边栏联动

#### 2.1 片段 hover → Transcript 高亮

- [ ] `PlaybackDetailTabs` 维护 `hoveredClipId` 状态
- [ ] 侧边栏片段列表项 onMouseEnter/Leave → `hoveredClipId`
- [ ] `hoveredClipId` 传入 `TranscriptView` → 对应时间范围的词高亮为琥珀色背景
- [ ] **反向**：transcript 中片段播放按钮 hover → sidebar 对应片段项高亮

#### 2.2 片段点击 → 跳播 + 滚动定位

- [ ] 点击侧边栏片段 → `playClip` + `onSeek(clip.startTime)` 确保 transcript 滚动到对应区域
- [ ] 播放中的片段在侧边栏高亮（匹配 `audioStore.mode` 的 clip id）

#### 2.3 听写记录点击 → 滚动到句子

- [ ] 点击听写记录 → `onSeek(sentence.start)` 跳转到句子开始时间
- [ ] 对应句子短暂高亮闪烁（新增 `flashSentence` prop 传入 TranscriptView）
- [ ] 当前高亮的句子与侧边栏选中的听写记录同步

#### 2.4 收藏项点击 → 定位

- [ ] 收藏的单词点击 → 跳转到该词在 transcript 中的第一个出现位置
- [ ] 收藏的片段点击 → 同 2.2

---

### 🟡 P1 — 片段颜色 & 播放状态可视化

#### 3.1 片段颜色选择

- [ ] **DB**: clips 表新增 `color` 列，默认 `#facc15`（琥珀色），含向后兼容迁移
- [ ] **API**: `ClipCreate` + `INSERT` 支持 `color` 字段
- [ ] **Transcript 工具栏**: 选中文本后弹出多个颜色小圆圈（红/橙/黄/绿/蓝/紫），点击切换当前片段颜色
- [ ] **Transcript 渲染**: 片段背景色使用对应 color 值，不再硬编码琥珀色
- [ ] **前端类型**: `AudioClip` 新增 `color` 字段，clipsStore CRUD 透传

#### 3.3 当前片段播放指示

- [ ] 当 `audioStore.mode.kind === 'clip'` 时，侧边栏对应片段项显示播放中动画（左侧绿色脉冲条）
- [ ] 片段循环时显示剩余次数（如「×3 剩余 2」）

#### 3.4 循环句子指示

- [ ] 当 `loopMode === 'sentence'` 时，transcript 中当前循环句显示循环图标
- [ ] 侧边栏 tabs 下方显示当前循环模式状态

#### 3.5 进度同步

- [ ] 侧边栏顶部显示当前播放时间的小型进度指示（仅当前 tab 有内容时）

---

### 🟢 P2 — 响应式 & 手势

#### 4.1 移动端侧边栏

- [ ] 宽度 < 1024px 时，侧边栏折叠为底部 tab bar
- [ ] 点击 tab 弹出 bottom sheet 面板（类似 LessonDetailPanel 的移动端模式）
- [ ] 面板内可正常浏览片段/听写/收藏

#### 4.2 侧边栏折叠按钮

- [ ] 桌面端侧边栏添加折叠/展开按钮
- [ ] 折叠后仅显示 tab 图标（无文字），宽度压缩至 48px
- [ ] 展开时 360px，带过渡动画
