# 产品计划 — 英语听力 App `v0.0.5`

> 继承自 [v0.0.4](../v0.0.4/product-plan.md) · 基于 [v0.0.4 评审报告](../v0.0.4/review-report.md)
>
> **v0.0.5 主题：后端完善 + 课程导入 + 进度统计**

## 目标

补齐后端 API 短板，上线课程导入 UI，添加学习进度统计，打磨错误处理和用户体验。

## v0.0.5 范围

### 🔴 P0 — 必须交付

#### 1. 单词 API 后端化

> **问题**：单词面板首次打开需 fetch 全部课程 JSON（30+ 节课时明显卡顿）

- [ ] **`GET /api/words`** — 返回去重单词列表
  - 参数：`?sort=freq`（按频率）、`?sort=alpha`（按字母）、`?limit=100`（分页）
  - 返回：`{ word, count, lessons: [{id, title, occurrences: [{start}]}] }`
- [ ] **单词面板改为按需加载** — 先显示高频词前 100 个，滚动加载更多
- [ ] **搜索改为后端搜索** — `?q=keyword`
- [ ] **移除前端全量 fetch** — WordsView 不再遍历全部课程 JSON

#### 2. 课程导入 UI

> **用户故事**：我想在 App 里直接上传音频文件，自动生成带词对齐的课程。

- [ ] **导入入口**：首页/课程页顶部「+ 导入课程」按钮
- [ ] **上传界面**：
  - 拖拽 MP3/WAV 文件到上传区域
  - 或点击选择文件
- [ ] **元数据表单**：
  - 标题（必填）
  - 副标题
  - 难度等级（A1-C1 下拉）
  - 来源 URL（选填）
- [ ] **后台处理流程**：
  ```
  上传完成 → 转写中 (WhisperX) → 对齐中 → 生成 JSON → 完成
  ```
- [ ] **进度指示**：底部 Toast 或进度条显示处理状态
- [ ] **完成后自动刷新**：课程列表和侧边栏计数更新

#### 3. 学习进度统计

> **用户故事**：我想看到自己的学习数据和进步趋势。

- [ ] **新增「统计」导航项**（替换或增强「最近播放」）
- [ ] **统计指标**：
  - 总听力时长（分钟）
  - 已完成课程数
  - 听写总句数 + 平均正确率
  - 收藏片段总数
  - 掌握单词数（在单词面板中标记"已掌握"的）
- [ ] **简单趋势图**：
  - 最近 7 天/30 天听力时长柱状图
  - 听写正确率折线图
- [ ] **存储**：localStorage + 可选后端持久化

### 🟡 P1 — 应该交付

#### 4. 全局错误处理

- [ ] **API 请求失败 Toast**：红色错误提示 + 错误信息
- [ ] **重试机制**：失败时显示「重试」按钮
- [ ] **网络状态检测**：断网时顶部显示离线横幅
- [ ] **错误边界**：React ErrorBoundary 防止白屏

#### 5. 听写模式增强

- [ ] **重新输入**：feedback 阶段增加「重新输入」按钮
- [ ] **跳过按钮**：不确定时可以跳过当前句
- [ ] **片段听写**：片段模式下也支持听写
- [ ] **快捷键优化**：Enter 提交，Ctrl+Enter 跳过

#### 6. 设置面板扩展

- [ ] **默认循环次数**：1-10 次可选
- [ ] **默认播放速度**：0.5x-1.5x
- [ ] **主题**：暗色/亮色切换（为未来准备）
- [ ] **快捷键提示**：列出所有键盘快捷键

#### 7. 骨架屏

- [ ] **首页骨架屏**：统计卡片 + 课程网格占位
- [ ] **单词面板骨架屏**：标签云占位
- [ ] **课程页骨架屏**：网格占位

## v0.0.4 → v0.0.5 对比

| 维度 | v0.0.4 | v0.0.5 |
|---|---|---|
| 单词加载 | 前端全量 fetch | 后端 API + 分页 |
| 课程导入 | 手动脚本 | UI 上传 + 自动处理 |
| 进度追踪 | 无 | 统计页面 + 趋势图 |
| 错误处理 | 静默失败 | Toast + 重试 + 离线提示 |
| 听写模式 | 基础 MVP | 重新输入 + 跳过 + 片段支持 |
| 设置 | 单词偏移 | 循环次数 + 速度 + 主题 |
| 加载体验 | 文字"加载中" | 骨架屏 |
