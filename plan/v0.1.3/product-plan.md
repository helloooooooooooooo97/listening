# v0.1.3 产品计划 — 播放队列重做

**日期**: 2026-06-07
**主题**: 播放队列从「能存」升级为「智能播放管线」，成为播放详情页的核心交互枢纽。

---

## 一、现状问题

当前播放队列：
- 只支持手动添加音频/片段
- 添加操作藏在播放栏一个不起眼的 ⊕ 按钮里
- 点击侧边栏片段 → 直接播放，不加入队列，打断当前播放流
- 听写句子无法加入队列
- 单词无法加入队列（带上下文）
- `playNext()` 只在 `ended` 时自动触发，手动点击片段不走队列

**用户期望**：点击任何内容（片段/听写/单词）→ 加入队列并顺序播放，而不是打断当前播放。

---

## 二、队列模型扩展

### 2.1 QueueItem 类型扩展

```ts
type QueueItem =
  | { kind: 'lesson'; lesson: ListeningLesson }
  | { kind: 'clip'; clip: AudioClip; lesson?: ListeningLesson | null }
  | { kind: 'sentence'; lessonId: string; lessonTitle: string; sentenceIndex: number; start: number; end: number; text: string }
  | { kind: 'word'; lessonId: string; lessonTitle: string; word: string; start: number; end: number }
```

- `sentence`: 包含句子起止时间和文本，播放时自动 seek + 句子循环
- `word`: 包含单词时间 + 前后偏移量（从设置读取），播放时创建临时 clip

### 2.2 队列行为

- 点击任何可播放项 → **追加到队列末尾 + 跳到该项开始播放**（不打断当前项，而是 cut 到该项）
- 队列为空时点击 → 该项成为队列第一项并开始播放
- `ended` 事件 → 自动播放下一项（已支持，需扩展支持新类型）
- 队列项播放完毕后自动移除（或保留在历史中）
- 当前播放项在队列中高亮

## 三、播放详情页集成

### 3.1 片段 → 智能队列

- [ ] 侧边栏每个片段项右侧添加「➕ 加入队列」按钮
- [ ] 右键片段 → 「加入队列」/ 「播放此片段」（加入队列并跳转）
- [ ] 「全部加入队列」按钮 — 将当前排序的所有片段一次性加入队列（去重）
- [ ] 点击片段默认行为：加入队列末尾 → 跳转到该片段开始播放（不再直接 playClip 打断）

### 3.2 听写记录 → 队列

- [ ] 每条听写记录右侧添加「➕」按钮 → 将该句加入队列（sentence 类型）
- [ ] 「复练全部错句」→ 将低于 80% 的句子批量加入队列
- [ ] 加入队列后自动 seek 到第一个错句开始播放

### 3.3 单词 → 上下文队列

- [ ] 单词 tab 中每个单词右侧「➕」→ 将该词加入队列（word 类型）
- [ ] 加入队列时带上 wordPlayOffset 上下文（前后各 N 秒）
- [ ] 「加入全部生词」→ 将未掌握的单词批量加入队列
- [ ] 播放单词队列项时：创建临时 clip，播放该词 + 上下文

## 六、实现要点

### 6.1 Store 改动
- `playlistStore`: 扩展 `QueueItem` 类型，新增 `playNow(item)` 方法（清空当前队列并播放）
- `audioStore`: `ended` 处理器支持 `sentence` 和 `word` 类型

### 6.2 播放 `sentence` 类型
- 设置 mode 为临时 clip（句子起止时间）
- 句子播完后自动切换到队列下一项

### 6.3 播放 `word` 类型
- 从 settings 读取 `wordPlayOffset`
- 创建临时 clip：`start = word.start - offset, end = word.end + offset`
- 播完后自动切换到下一项
