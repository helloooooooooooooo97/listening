# v0.0.1 回顾评审

> 评审日期：2026-06-06 · 类型：事后回顾

## 总评

v0.0.1 完成了核心基础设施搭建，但前端体验粗糙，状态管理缺失导致多个Bug。

## 做得好的

1. **WhisperX管线** — 一键生成词对齐JSON，产出质量可接受
2. **后端API** — RESTful设计清晰，3个端点覆盖核心需求
3. **课程导入检查清单** — 文档化操作流程，降低上手门槛
4. **Makefile** — 一键启动前后端，开发体验好

## 主要问题（在v0.0.2中已修复）

| 问题 | v0.0.2 修复方案 |
|---|---|
| 播放状态不同步 — 侧边栏和播放器各自管理 | Zustand全局audioStore |
| 片段收藏失效 — useClips多组件状态隔离 | Zustand clipsStore |
| 页面跳转过多 — 列表→播放器→片段→复习四层 | 统一左右分栏布局 |
| 无全局播放控制 — 切换页面时音频中断 | 底部固定PlayerBar + 全局Audio单例 |

## 遗留到v0.0.2的问题

- 精听/听写/跟读模式均未实现 → v0.0.2仍未实现
- UI设计粗糙 → v0.0.2 Notion风格重构
- 无全局状态管理 → v0.0.2 Zustand引入

## 结论

v0.0.1的架构选型（React+FastAPI+WhisperX）正确，数据管线扎实。主要短板在前端工程质量，v0.0.2通过引入Zustand和统一布局解决了核心架构问题。
