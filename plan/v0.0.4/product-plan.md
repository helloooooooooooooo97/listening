# 产品计划 — 英语听力 App `v0.0.4`

> 继承自 [v0.0.3](../v0.0.3/product-plan.md) · 基于 [v0.0.3 评审报告](../v0.0.3/review-report.md)
>
> **v0.0.4 主题：听写模式 + 工程质量**

## 目标

补齐听写模式，拆分巨型组件，加入课程导入 UI，打磨动画细节。

## v0.0.4 范围

### 🔴 P0 — 必须交付

#### 1. 听写模式 MVP

> **用户故事**：我想听完一句话后输入听到的内容，系统告诉我哪里写对了、哪里写错了。

- [ ] **模式切换**：PlayerBar 右侧增加「听写」按钮，点击进入听写模式
- [ ] **隐藏文本**：进入听写模式后，文本区隐藏，仅显示当前句子的占位提示
- [ ] **逐句播放**：自动播放当前句子，不自动进入下一句
- [ ] **输入框**：每句播放后显示输入框，用户输入听到的内容
- [ ] **对比反馈**：提交后逐词对比原文：
  - ✅ 正确词 → 绿色
  - ❌ 错误/遗漏词 → 红色
  - ⚠️ 多余词 → 黄色删除线
- [ ] **分数**：每句完成后显示正确率（正确词数/总词数）
- [ ] **下一句**：手动点击进入下一句

#### 2. ContentPanel 拆分

> **技术债**：当前 ContentPanel.tsx 超过 500 行，包含 5 个视图。

- [ ] 拆分为独立组件：
  - `HomeView.tsx` — 首页仪表盘
  - `CoursesView.tsx` — 课程网格
  - `ClipsView.tsx` — 片段列表
  - `WordsView.tsx` — 单词面板
  - `RecentView.tsx` — 最近播放
- [ ] ContentPanel 改为路由分发器（根据 activeSection 渲染对应组件）
- [ ] 共享逻辑提取为 hooks（`useSearch`、`useStats`）

#### 3. 过渡动画

- [ ] **歌词覆盖层**：展开 slide-up (0.4s)，收起 slide-down (0.3s)
- [ ] **视图切换**：activeSection 切换时 fade-in 过渡
- [ ] **听写反馈**：词对比结果逐词弹出动画

---

### 🟡 P1 — 应该交付

#### 4. 课程导入 UI

> **用户故事**：我想在 App 里直接上传音频，自动生成课程，不需要手动运行脚本。

- [ ] **上传界面**：首页增加「导入课程」按钮
- [ ] **拖拽上传**：支持拖拽 MP3 文件到上传区域
- [ ] **元数据表单**：标题、副标题、难度、来源 URL
- [ ] **后台处理**：上传后调用 WhisperX（异步任务）
- [ ] **进度指示**：显示处理进度（上传中 → 转写中 → 对齐中 → 完成）
- [ ] **完成后刷新**：自动加入课程列表

#### 5. 学习进度持久化

> **用户故事**：我想看到自己学了多久，完成了多少课程。

- [ ] **统计页面**：新增「统计」导航项
- [ ] **数据指标**：
  - 总听力时长
  - 已完成课程数
  - 听写正确率趋势
  - 收藏片段数
  - 单词掌握数
- [ ] **存储**：localStorage + 可选后端持久化

#### 6. 单词面板优化

- [ ] **后端单词 API**：`GET /api/words` 返回去重单词列表 + 频率 + 所在课程
- [ ] **移除前端全量加载**：不再 fetch 全部课程 JSON
- [ ] **分页/虚拟滚动**：单词超过 500 个时虚拟滚动

#### 7. 错误处理

- [ ] **API 错误 Toast**：fetch 失败时显示错误提示
- [ ] **重试按钮**：加载失败时显示重试
- [ ] **网络状态检测**：断网时显示离线提示
