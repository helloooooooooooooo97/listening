## 二、功能评审

### ✅ v0.0.4 已交付

1. **ContentPanel 拆分** — 500+ 行 → 30 行 dispatcher + HomeView/CoursesView/ClipsView/WordsView/RecentView/DictationView/SettingsView 7 个独立组件
2. **听写模式 MVP** — 逐句自动播放→停止→输入→逐词红绿反馈→评分，流程流畅
3. **单词面板增强** — 去重+频率排序+每词出现时间戳列表+按课程分组
4. **单词片段播放** — 点击时间戳以片段模式播放（±offset秒），紫色图标+单词标题
5. **设置系统** — 侧边栏底部设置入口，单词播放偏移量可调（0.5s–5s），localStorage持久化
6. **播放栏简化** — 删除-5s/+5s按钮，只保留上句/播放/下句三按钮
7. **播放栏排版修复** — 不再覆盖侧边栏（left-52），只在右侧内容区
8. **单词清洗** — 前后端同步去除,.!等标点符号，去重统计准确
9. **片段歌词覆盖** — 片段播放时可展开完整课程TranscriptView
10. **加载防抖** — 播放按钮不再因seek闪烁（200ms debounce）
11. **键盘快捷键** — ←→句子导航，Shift+←→时间跳跃
12. **进度条全音频** — 片段播放时显示完整音频进度+片段高亮范围

### 🐛 已知问题

1. **单词面板首次加载慢** — 需 fetch 全部课程 JSON，课程多时明显
2. **听写模式仅支持课程** — 片段模式下无法听写
3. **单词时间戳播放后无自动停止** — 依赖片段循环模式，需手动设置循环次数
4. **设置面板功能单一** — 仅单词偏移一项
5. **无撤销/重做** — 听写提交后无法修改

## 三、UI/UX 评审

### ✅ 做得好的

4. **统计卡片 5 列布局** — 首页信息密度合理

## 四、架构评审

### ✅ 优点

1. **View 组件拆分** — 7 个独立 View，每个 < 200 行，职责单一
2. **Store 分层** — audio/clips/settings/dictation/toast 5 个 Store 分工明确
3. **防抖加载** — 200ms debounce 解决 seek 闪烁，优雅
4. **单词清洗** — 正则去除标点，前后端一致
5. **settingsStore 扩展** — 在原有播放记忆基础上增加 AppSettings

### ⚠️ 风险

1. **无单词 API** — `GET /api/words` 仍未实现，前端全量 fetch
2. **dictationStore 耦合** — 听写逻辑依赖 audioStore 的 mode.kind===lesson
3. **无测试** — 0 单元测试，回归依赖手动

## 五、v0.0.5 建议

## 六、评审结论

v0.0.4 完成了组件拆分和听写模式两个核心目标，工程质量显著提升。单词面板的时间戳播放和设置系统是超出预期的亮点。**主要短板仍在后端——单词全量加载和课程导入流程需后端化。**

**建议 v0.0.5 聚焦：后端 API 完善 + 课程导入 UI + 进度统计**。
